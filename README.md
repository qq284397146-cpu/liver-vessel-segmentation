## 🔧 交付保障（工程化实践）

**能独立解决的典型生产环境问题**：
- 模型权重缺失/路径配置（nnUNet Dataset编号与checkpoint管理）
- License配置报错（CUDA/显存限制、虚拟内存不足）
- 内存优化：扩虚拟内存 + Slicer与AI模型分离运行（避免互相抢资源）

**标准化任务流程（防踩坑）**：
1. **环境验证**：接到任务先在**小数据集试跑**，确认TotalSegmentator环境无误再正式开工
2. **内存管理**：跑AI模型时关闭Slicer，用cmd单独跑TotalSegmentator，跑完再导入Slicer修正
3. **进度保存**：每完成一个结构就导出一次（.mrb/.nii），不等到全部做完再保存，防止崩溃丢工作
4. **批量处理**：107层CT序列稳定处理，支持DICOM无扩展名鲁棒性读取

## 🐍 Python数据处理工具

### DICOM元数据批量提取器
**文件位置**：`tools/dicom_extractor.py`

**解决的问题**：医院导出的DICOM通常无扩展名（纯数字命名），手工查看107层CT元数据效率极低。

**功能**：
- 自动识别无扩展名DICOM文件
- 批量提取：PatientID、StudyDate、SliceThickness、Rows、Columns
- 输出标准化CSV报表

**验证规模**：单次处理107层CT序列（完整病例）
